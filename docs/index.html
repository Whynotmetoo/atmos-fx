React Integration

Use the modern, type-safe React components to seamlessly integrate weather effects and interactive cards into your React application:

import { AtmosFx, AtmosCard } from 'atmos-fx'

export function WeatherPanel() {
  return (
    <AtmosFx mode="rain" density={1.0}>
      {/* Normal elements inside the wrapper */}
      <button>Upgrade to Pro</button>
      
      {/* Interactive cards wrapped with AtmosCard */}
      <AtmosCard transMode="glass" liquidDripping={true}>
        <div>Glass card content</div>
      </AtmosCard>
      
      {/* Use asChild to merge styling and attributes onto custom child elements directly */}
      <AtmosCard asChild transMode="solid">
        <button className="custom-btn">Opaque Action Button</button>
      </AtmosCard>
    </AtmosFx>
  )
}

API Reference

Option Type / Values Description
preset 'rain' | 'snow' | 'hail' Applies preset default physical and visual values.
particle 'rain' | 'snow' | 'hail' Overrides preset particle rendering without overwriting speed/wind presets.
density 0 to 1 0.65 Scales preallocated particle budget and spawn count.
speed 0 to 10 1 Scalar multiplier for gravity and vertical fall speed.
wind -1.5 to 1.5 -0.12 Affects horizontal sway and particle drift.
color string Ice Blue CSS color representation for precipitation particles.
quality 'auto' | 'low' | 'medium' | 'high' Limits particle count and device pixel ratio density. Defaults to auto.
bottomCollision boolean false Determines whether particles collide with the bottom edge of the container.
transparency 'glass' | 'opacity' | 'none' Integrates elements with the weather background via CSS blur or opacity filter.
surfaceOpacity 0 to 1 0.14 Backdrop opacity multiplier used in glass transparency mode.
contentOpacity 0 to 1 0.72 Foreground text readability fading used in opacity transparency mode.
snowAccumulation 0 to 1 0.55 BUildup intensity of snow piles on top of collision targets.
hailBounce 0 to 1 0.5 Coefficient of restitution for bouncing hail pellets.
collisionSelector string Query selector for discovering top-edge landing surfaces. Defaults to [data-atmos-collision].
Note: Rotated elements (e.g. transform: rotate(...)) calculate collisions against their axis-aligned bounding box (AABB) rather than the rotated visual boundary.
opaqueSelector string Query selector for elements that skip transparency blurs. Defaults to [data-atmos-opaque].

Controller Methods

Method Description
start() Initializes canvas layers, starts the requestAnimationFrame loop and rendering.
stop() Stops the animation loop and clears canvases. Does not remove layers.
pause() Pauses the animation execution while keeping the active frame buffer.
resume() Resumes execution from a paused state.
resize() Manually triggers context bounds measurement and canvas resizing.
update(options) Dynamically updates atmosphere parameters on the fly without resetting state.
destroy() Cleans up DOM nodes, listeners, context properties, and cancels loops.

DOM Attributes

Attribute Description
data-atmos-collision Marks the element as a top-edge landing surface for snow, rain splashes, and hail.
Note: Rotated elements calculate collisions against their outer bounding box rather than the rotated edge.
data-atmos-opaque Instructs the glass controller to keep this element solid without translucent styling.
data-atmos-glass Explicitly opts nested panels/containers into receiving the default weather-blur visual filter.
data-atmos-opacity="0.65" Specifies an element-specific glass background opacity level.
